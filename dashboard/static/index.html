Deploy Agent

      
Run the VectorBridge agent on your server. Vectors never leave your network.


      

        
Option A — Docker (Recommended)

        

          Paste this on any server where your source vector database is accessible.
          The agent fetches its config from this dashboard automatically.
        

        

          
          
# Run once (full migration)
docker run --rm \
  --network host \
  -e VB_LICENSE_KEY=YOUR_KEY \
  -e VB_JOB_ID=YOUR_JOB_ID \
  vectorbridge/agent:latest

# Continuous sync (incremental every hour)
docker run -d \
  --name vectorbridge-agent \
  --network host \
  --restart unless-stopped \
  -e VB_LICENSE_KEY=YOUR_KEY \
  -e VB_JOB_ID=YOUR_JOB_ID \
  -e VB_POLL_SECONDS=3600 \
  vectorbridge/agent:latest

        

      


      

        
Option B — Kubernetes

        

          Deploy as a K8s Job (one-time) or CronJob (scheduled sync).
        

        

          
          
# Apply the secret (replace values first)
kubectl apply -f k8s/secret.yaml

# One-time migration job
kubectl apply -f k8s/deployment.yaml

# Hourly incremental sync
kubectl apply -f k8s/cronjob.yaml

# Watch logs
kubectl logs -f -n vectorbridge -l app=vectorbridge-agent

        

        

          Download K8s manifests from
          
            github.com/insightitsGit/vectorbridge/k8s
          
        

      


      

        
Option C — Python Library

        

          
          
pip install vectorbridge

# Run from config file
vectorbridge run --config vb_config.json

# Or Python API
from vectorbridge import Bridge
bridge = Bridge.from_config("vb_config.json")
report = bridge.run()
print(report.summary())
